# Skills

Skills are reusable capabilities that agents can apply to repository work.

## Core Skills

| Skill | Use |
| --- | --- |
| Repository Governance | Catalog, ownership, 13 pillars and project lifecycle. |
| API Contracts | OpenAPI, Swagger UI readiness and Postman collections. |
| MCP | Client/server integration, tool/resource/prompt boundaries and auth. |
| RAG | Corpus, chunking, indexing, retrieval, evaluation and privacy. |
| Documentation | READMEs, ADRs, RFCs, diagrams, badges and style. |

## Skill Definition Rule

Each skill must define:

- When to use it.
- Inputs required.
- Outputs expected.
- Validation command or manual evidence.
- Security and privacy constraints.
