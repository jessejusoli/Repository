# Contracts

This directory stores language-neutral contracts for APIs, integrations and AI
systems.

## Areas

- `openapi/`: HTTP API descriptions using OpenAPI.
- `postman/`: Postman collections for API workflows and examples.
- `mcp/`: MCP client/server registry and integration expectations.
- `rag/`: RAG system profiles and evaluation expectations.

## Rule

Contracts should be usable before implementation details are known. They define
interfaces, boundaries and validation evidence, not a specific framework.
