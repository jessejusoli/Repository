# Repository Baseline

## Objetivo

Orientar quando e como aplicar `repository-baseline` sem copiar a documentacao oficial para dentro do Repository.

## Quando Aplicar

Aplica-se a: all-projects.

## Criterios De Adocao

- Project has README, ownership, validation, contribution and security guidance.
- Required repository artifacts are tracked and verified by CI.

## Artefatos Esperados

- `README.md`
- `CONTRIBUTING.md`
- `SECURITY.md`
- `CODE_OF_CONDUCT.md`
- `CHANGELOG.md`
- `.editorconfig`
- `.gitattributes`
- `.gitignore`

## Como Usar No Repository

- Consulte este pocket guide para decidir aplicabilidade.
- Registre evidencia no README do projeto, catalogo, contrato ou checklist correspondente.
- Use o link oficial para detalhes completos, exemplos extensos e mudancas de versao.
